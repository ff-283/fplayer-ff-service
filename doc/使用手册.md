# 使用手册（service）

本文档只讲“怎么用”。

## 1. 适用场景

- 你希望 `desktop` 与 `service` 分机部署（不同电脑）
- 推流端先向 service 申请会话，再开始推流
- 观看端通过 service 提供的地址拉流（HTTP-FLV 或 RTMP）

## 2. 启动 service（Windows）

在 `fplayer-ff-service` 根目录执行：

```powershell
.\scripts\start-all.ps1
```

说明：

- 脚本会自动启动 ZLM + gateway + UI，并写入 `run/runtime.json`
- 默认优先使用 `9000` 作为 gateway 端口；若冲突会自动切换到空闲端口
- 子进程控制台窗口默认隐藏，不会弹出多个黑框

停止：

```powershell
.\scripts\stop-all.ps1
```

### 2.1 发布包启动（推荐）

若你拿到的是发布包（`release/portable`），直接双击：

- `FPlayerFFService.exe`

说明：

- 不需要手动执行 `start-all.ps1`
- 建议等待 UI 状态显示“服务已就绪”后，再在 desktop 操作“创建并开始”

## 3. 启动后检查

### 3.1 健康检查

浏览器打开：

- `http://127.0.0.1:<gateway-port>/healthz`

返回：

```json
{"status":"ok"}
```

### 3.2 查看运行端口

查看文件：

- `run/runtime.json`

重点字段：

- `gateway.url`：desktop 里要填写的“服务地址”
- `endpoints.publishRtmp`：媒体上行地址（由 service 返回，不需要手填）
- `endpoints.playHttpFlv`：播放地址示例

## 4. desktop 如何填写（服务端模式）

desktop 推流窗口：

1. `推流模式` 选 `推送服务端`
2. `服务地址` 填 `run/runtime.json` 中的 `gateway.url`（跨机器时将 `127.0.0.1` 换成 service 实际 IP）
3. 填 `服务 app`（如 `live`）
4. 填 `服务 stream`（如 `stream001`）
5. 选 `服务类型`：
   - `broadcast`：广播分发
   - `httpflv`：广播分发 + HTTP-FLV 优先
6. 点击 `开始推流`

> 注意：`服务地址` 是 service 机器地址，不是 desktop 本机地址。

## 5. 拉流方式

### 5.1 服务端拉流（推荐）

拉流窗口：

1. `拉流模式` 选 `服务端拉流`
2. 填 `服务地址` 和 `streamId`
3. 选优先协议（HTTP-FLV/RTMP）
4. 点 `解析地址` 或直接 `开始拉流`

### 5.2 P2P 拉流

拉流窗口：

1. `拉流模式` 选 `P2P 拉流`
2. 选择协议模板并填写推流码
3. 走本地拼接地址方式

## 6. 跨机器部署清单

- service 机器放行端口：
  - gateway：`run/runtime.json` 中 `gateway.port`（默认优先 `9000`）
  - ZLM 实际 RTMP/HTTP 端口（见 `run/runtime.json`）
- desktop 机器可访问 service 的 `gateway.port`
- 若跨网段，确保路由/NAT/防火墙策略放通

## 7. 常见报错处理

### 7.1 `创建失败: Failed to fetch`

检查：

- `http://<service-ip>:<gateway-port>/healthz` 是否可达（`gateway-port` 以 `run/runtime.json` 为准）
- gateway 是否仍在运行（看 `logs/gateway*.log`）
- desktop 的服务地址是否写对

### 7.2 UI 没有弹出

检查：

- `ui/node_modules` 是否安装完整
- `logs/ui.err.log` 是否有 `electron` 下载或命令错误
