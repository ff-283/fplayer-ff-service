# 打包发布与使用教程（Windows）

本文档面向发布人员与最终使用者，说明如何生成可分发包，以及用户拿到包后如何直接运行。

## 1. 发布目标

发布结果支持两种目录形态：

- `release/portable-ui`：仅 UI 可执行程序
- `release/portable-kernel`：仅 Kernel 可执行程序
- 运行依赖均从 exe 同级目录查找
- 不需要用户手动执行 `start-service.bat` 或 `scripts/start-all.ps1`

## 2. 发布前准备

在 `fplayer-ff-service` 项目根目录确认：

- 已安装 Go（用于编译 `gateway.exe`）
- 已安装 Node.js/npm（用于打包 Electron）
- `3rd/zlm/windows/MediaServer.exe` 存在
- `3rd/zlm/windows/config.ini` 存在

## 3. 一键打包发布

在项目根目录执行：

```powershell
.\scripts\build-release.ps1
```

脚本会自动完成：

1. 编译 `gateway/bin/gateway.exe`
2. 打包 Electron（生成 `ui/dist/win-unpacked` 与安装包 exe）
3. 生成 `release/portable-ui` 与 `release/portable-kernel`（分离分发目录）
4. 校验关键依赖是否齐全，缺失则直接失败

### 3.1 清理后重打包（可选）

如果你希望先清理构建产物再发布，可先执行：

```powershell
.\scripts\clean.ps1
```

然后再执行：

```powershell
.\scripts\build-release.ps1
```

`build-release.ps1` 会重新生成发布需要的全部可再生产物（前提是 `3rd/` 等核心依赖仍在）。

## 4. 产物说明（release）

打包成功后，会看到：

- `release/portable-ui/`（仅 UI 可执行程序）
- `release/portable-kernel/`（仅 Kernel 可执行程序）
- `release/win-unpacked/`（原始解包产物）
- `release/FPlayerFFService 0.1.0.exe`（安装包/便携单文件产物）

推荐按目标人群分发：

- 需要可视化操作：分发 `release/portable-ui`
- 需要无人值守运行：分发 `release/portable-kernel`
- 需要同时支持两种方式：同时分发 `release/portable-ui` 与 `release/portable-kernel`

## 5. 如何分发给用户

推荐方式：

1. 按需求选择目录并整体打包（zip）：
   - `release/portable-ui`
   - `release/portable-kernel`
2. 发给用户并解压到任意目录
3. 用户按目标程序双击启动：
   - UI：`FPlayerFFService.exe`
   - Kernel：`FPlayerFFServiceKernel.exe`

注意：不要只发单个 exe，必须连同同级依赖目录一起发。

## 6. 用户侧使用教程

### 6.1 启动服务

双击以下其一：

- `FPlayerFFService.exe`（UI 模式）
- `FPlayerFFServiceKernel.exe`（内核模式，无界面）

两种模式都会自动在后台拉起 service 内核（ZLM + gateway）。

### 6.2 等待“服务已就绪”

UI 模式会显示启动状态：

- `正在初始化服务...` / `正在启动 ZLM 与 gateway...`：启动中
- `服务已就绪（gateway: ...）`：可开始创建流
- `启动失败：...`：需查看日志排障

Kernel 模式会在命令行打印：

- 当前状态心跳
- `gateway` 地址
- 推流/拉流端点信息

### 6.3 在 desktop 里填写服务地址

以运行后生成的文件为准：

- `run/runtime.json` 的 `gateway.url`

把这个地址填写到 desktop 的“服务地址”字段。

## 7. 最小自检清单（发布前）

分离分发目录的最小可执行程序要求为：

- `release/portable-ui` 必须包含 `FPlayerFFService.exe`
- `release/portable-kernel` 必须包含 `FPlayerFFServiceKernel.exe`

## 8. 常见问题

### 8.1 点击“创建并开始”报 `Failed to fetch`

优先检查：

- UI 状态是否已到“服务已就绪”
- `run/runtime.json` 是否已生成
- `logs/gateway.err.log` 是否有启动错误
- desktop 填写的是否是 `runtime.json` 中的 `gateway.url`

### 8.2 提示缺少 `ffmpeg.dll`

说明分发包不完整。请重新执行 `build-release.ps1`，并分发完整的 `release/portable-ui` 或 `release/portable-kernel` 目录。

### 8.3 端口冲突

当前版本会自动选择可用端口，实际端口以 `run/runtime.json` 为准。
