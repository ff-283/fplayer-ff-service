# FPlayer Service 用户手册

本文档面向最终用户，说明 `fplayer-ff-service` 如何与 `fplayer-ff-desktop` 配合使用。

## 1. service 是做什么的

`fplayer-ff-service` 提供：

- 会话创建与地址编排（gateway API）
- 媒体分发（依赖内置 ZLMediaKit）
- 多端观看支持（广播/协议分发）

它不是推流采集端，采集和推流入口仍在 `desktop`。

## 2. 启动方式（发布包）

推荐按需求选择发布目录：

1. UI 模式：双击 `portable-ui/FPlayerFFService.exe`
2. Kernel 模式：双击 `portable-kernel/FPlayerFFServiceKernel.exe`
3. UI 模式下等待“服务已就绪（gateway: ...）”；Kernel 模式下查看控制台输出（支持 `Ctrl+C` 停止）

运行后会生成：

- `run/runtime.json`

关键字段：

- `gateway.url`：desktop 应填写的服务地址
- `gateway.port`：实际网关端口
- `endpoints.*`：示例推/拉流地址

同时，service UI 可直接复制以下字段：

- Gateway URL
- App / Stream
- RTMP 推流地址
- HTTP-FLV 播放地址
- mobile 拉流地址（playHttpFlv）

## 3. 如何配合 desktop（核心流程）

### 3.1 desktop 推流端

在 desktop 推流窗口：

1. 选择“推送服务端”
2. 服务地址填 `gateway.url`
3. 填写 `app`、`stream`
4. 服务类型选择：
   - `broadcast`：广播分发（RTMP + FLV + HLS）
   - `httpflv`：广播分发 + HTTP-FLV 优先（含 HLS）
5. 点击 `开始推流`

### 3.2 desktop 拉流端

在 desktop 拉流窗口：

1. 选择“从服务端按 streamId 自动解析”
2. 服务地址填 `gateway.url`
3. 输入 `streamId`
4. 优先协议建议 `HTTP-FLV`
5. 点击 `解析地址` 或 `开始拉流`

## 4. 广播模式怎么用（已支持）

当前广播使用方式：

1. service 启动并就绪
2. desktop 推流选择服务类型 `broadcast`
3. 观看端通过 service 返回的 `playUrls` 拉流

说明：

- 广播分发由 ZLMediaKit 媒体内核完成
- service 负责会话和地址管理
- 适合一对多观看场景

## 5. 部署建议

- 跨机器使用时，desktop 和 service 保持网络可达
- 不要写死 `:9000`，要以 `runtime.json` 实际端口为准
- 若防火墙开启，放行 `gateway.port` 及相关媒体端口

## 6. 常见问题

### 6.1 `Failed to fetch`

排查顺序：

1. service 是否已“服务已就绪”
2. `run/runtime.json` 是否存在
3. desktop 是否填写了正确 `gateway.url`
4. 查看日志：
   - `logs/gateway.err.log`
   - `logs/ui.err.log`

### 6.3 mobile 播放失败/端口不对

建议处理方式：

1. 不手写固定 `:8080`
2. 优先使用 service 返回的 `playHttpFlv`（UI 中“mobile 拉流地址”可直接复制）
3. 或在 mobile 中填写 `gateway.url + app + stream`，由 mobile 自动解析播放地址

原因：

- service 启动时会优先尝试 8080，但若被占用会自动切到空闲端口
- 实际端口以运行时为准，不应硬编码

### 6.2 提示缺少 DLL 或无法启动

通常是发布目录不完整。请确认使用完整发布目录，且包含：

- `FPlayerFFService.exe`
- `FPlayerFFServiceKernel.exe`（若分发内核版）
- `ffmpeg.dll`
- `resources/app.asar`
- `3rd/zlm/windows/MediaServer.exe`
- `gateway/bin/gateway.exe`

## 7. 推荐给用户的话术

对最终用户可简化为：

1. 启动对应程序（UI 或 Kernel）
2. 获取 gateway 地址（UI 状态区或 Kernel 控制台输出）
3. 把 gateway 地址填到 desktop 里
4. 推流选服务端模式，服务类型选 `broadcast`
